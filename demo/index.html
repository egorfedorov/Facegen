Why facegen?

Built for real projects

0

Zero dependencies

Pure SVG + CSS. No canvas, no runtime deps, no WebGL. Works everywhere from Node to the browser.

=

Deterministic

Same input string = same face. Always. No randomness, no server calls. Hash-based parameter extraction.

~

Animated

Eyes blink, faces float, mouths twitch. Pure CSS keyframes with unique phase offsets per avatar.

A

Accessible

Respects prefers-reduced-motion. Proper role="img" and aria-label on every avatar. Screen-reader friendly.

API

Props reference

Prop Type Default Description
seed string — Any string — email, username, UUID. Determines the face.
size number 64 Avatar size in pixels. Below 32px, animations are disabled.
animate boolean true Enable blinking, floating, and mouth twitching animations.
alt string "Avatar" Accessible label for the avatar (React only).
className string — CSS class for the wrapper element (React only).
style CSSProperties — Inline styles for the wrapper element (React only).

Usage

Code examples

React 
import { Facegen } from 'facegen';
import 'facegen/css';

function UserAvatar({ user }) {
  return <Facegen seed={user.email} size={48} animate />;
}
Vanilla JS 
import { generateFaceSvg } from 'facegen';

const svg = generateFaceSvg('alice@example.com', {
  size: 48,
  animate: true,
});

document.getElementById('avatar').innerHTML = svg;
img tag / data URI 
import { generateFaceSvg } from 'facegen';

const svg = generateFaceSvg('bob', { size: 64 });
const dataUri = `data:image/svg+xml,${encodeURIComponent(svg)}`;

// Use as <img> src (no animations, but works everywhere)
const img = new Image();
img.src = dataUri;
